# **App Name**: HearStart

## Core Features:

- Infant Registration: Collect and store infant demographic information including name, date of birth, gender, and contact details. Calculates age in months automatically.
- Hearing Screening Status: Record the infant's hearing screening status (Passed, Referred, Not Yet Screened) and provide recommendations based on the selected status. Generate recommendations for users via a tool to decide whether to remind them when additional screenings are necessary.
- High-Risk Register: Allow users to select risk factors from a pre-defined list and save them to Firestore.
- Auditory Behavioural Checklist: Present age-based milestones for auditory behaviour and allow users to check off completed milestones.
- Ling Six Sound Test: Play Ling Six sounds and environmental sounds and record user observations.
- User Authentication: Enable user authentication (email login) for parents/health workers.
- Data Analytics: Capture a Firebase Analytics event each time a screening is submitted.

## Style Guidelines:

- Primary color: Light blue (#B0E2FF), suggesting calmness and care, appropriate for the app's healthcare context.
- Background color: White (#FFFFFF), offering a clean and readable backdrop.
- Accent color: Soft Lavender (#E6E6FA), used to highlight key interactive elements, guiding users gently through the app.
- Main character color: Orange (#FFA500), used to highlight elements associated with the main character.
- Body and headline font: 'PT Sans' for a modern yet friendly feel.
- Use clear, intuitive icons for navigation and interactive elements, ensuring ease of understanding for all users.
- Employ a clean and simple layout with a navigation drawer or bottom navigation bar for easy access to different sections.
- Use subtle animations and transitions to enhance the user experience and provide feedback on interactions.
